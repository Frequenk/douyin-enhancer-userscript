# 数据统计功能方案（修订版）

## 目标
在现有控制面板最左侧新增一个**非开关**统计项（单行文字），用于展示：
- `今[数量][HH:MM:SS]`（例如：`今128 00:42:19`）

点击该统计项弹出统计面板（图表 + 导入/导出）。

---

## UI 交互草案
### 统计项（面板左侧）
- 形式：单行文字卡片，无开关。
- 文案示例：`今 128 00:42:19`
- 点击触发统计弹窗。

### 统计弹窗结构
顶部为“今日数据区域”，且**“今日”可切换范围**：
- 选项：`本日` / `本月` / `本年` / `所有`
- 该区域展示所选范围内的所有字段汇总值（视频数、时长、各类计数）

其余区域从上到下：
1. **年度热力图**（展示某一年每天的刷视频数量/时长）
   - 右侧提供年份选择器（下拉）
2. **每月刷视频数量柱状图**（受年份选择影响）
3. **每月刷视频时间柱状图**（受年份选择影响）

---

## 统计维度与口径（天粒度）
仅保留以下字段：
- `date`：日期（YYYY-MM-DD）
- `videoCount`：今日刷视频总数
- `watchTimeSec`：今日刷抖音总时长（秒）
- `skipLiveCount`：跳过直播次数
- `skipAdCount`：跳过广告次数
- `blockKeywordCount`：关键字屏蔽触发次数
- `aiLikeCount`：AI 自动点赞次数
- `speedSkipCount`：极速模式触发跳过次数

说明：
- **视频数**口径：每次检测到“新视频开始”时 +1。
- **时长**口径：只统计页面处于激活状态、视频正在播放时的累计时长（秒）。

---

## 数据存储（IndexedDB）
### DB 设计
- DB 名：`douyin-enhancer-stats`
- Version：1
- Store：`dailyStats`
- KeyPath：`date`

### 示例记录
```json
{
  "date": "2026-02-25",
  "videoCount": 128,
  "watchTimeSec": 2539,
  "skipLiveCount": 6,
  "skipAdCount": 3,
  "blockKeywordCount": 4,
  "aiLikeCount": 9,
  "speedSkipCount": 11
}
```

### 累计策略
- 内存中维护“今日统计缓存”，每次事件触发后合并写入 IndexedDB。
- 以 1~5 秒节流写入，避免频繁 IO。
- 每天首次进入时初始化当日记录。

---

## 统计触发点（事件来源）
- **videoCount**：检测到 `handleNewVideo()` 发生时 +1
- **watchTimeSec**：主循环中累积（仅在视频播放且页面可见时）
- **skipLiveCount**：调用 `videoController.skip('⏭️ 自动跳过: 直播间')` 时 +1
- **skipAdCount**：`checkAd()` 触发跳过时 +1
- **blockKeywordCount**：`checkBlockedAccount()` 命中关键字时 +1
- **aiLikeCount**：`videoController.like()` 时 +1
- **speedSkipCount**：`极速模式` 达到时间触发跳过时 +1

---

## 图表与筛选逻辑
### 年份选择器
- 影响范围：
  - 年度热力图
  - 每月刷视频数量柱状图
  - 每月刷视频时间柱状图

### 热力图
- 维度：按天
- 展示：选定年份的 1~12 月每天数据

### 月度柱状图（两张）
- `视频数量柱状图`：展示选定年份 1~12 月每月 `videoCount` 汇总
- `刷视频时长柱状图`：展示选定年份 1~12 月每月 `watchTimeSec` 汇总

---

## 导入与导出
### 导出
- 导出 JSON 文件，包含全量 `dailyStats`
- 可选：导出指定年份或区间

### 导入
- 读取 JSON 后写入本地数据库
- **冲突策略**：如果日期重复，以导入数据为准，覆盖本地存量
- 导入前弹窗提示：
  - `如有重复日期，将以导入数据为准，覆盖本地存量。`

---

## 技术实现建议
- 封装 `StatsStore`：
  - `init()`：打开 DB
  - `getDaily(date)` / `upsertDaily(record)`
  - `getRange(start, end)`
  - `exportAll()` / `importAll(records)`
- 封装 `StatsTracker`：
  - 内存缓存 + 节流落盘
  - 对外暴露 `inc(field, delta)` / `addTime(seconds)`
- UI 里只读统计值，不直接操作 DB。

---

## 风险与注意事项
- IndexedDB 写入失败时应降级（日志 + 临时内存）
- 统计时长需规避后台播放误计（仅 `document.visibilityState === 'visible'`）
- 不同浏览器脚本环境对 IndexedDB 支持差异较小，但需要异步处理。
